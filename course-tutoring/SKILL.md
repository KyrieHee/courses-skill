# Course Tutoring

## Workflow

1. Identify the exact confusion point and learner level.
2. Give a short plain-language explanation before formal terminology.
3. Build the concept or text in layers: intuition, structure, key details, example, transfer.
4. Show one worked example, close reading, proof sketch, or model answer when useful.
5. Name common mistakes and how to self-check.
6. End with a short learner task and feedback criteria.

## Tutoring Rules

- Do not overload: one main idea per explanation block.
- For K-12, use age-appropriate analogies and scaffolded questions.
- For university, include assumptions, definitions, notation, and links to prerequisites.
- For humanities, distinguish interpretation from evidence in the text.
- For STEM, distinguish formula memory from method selection and reasoning.

## Output

Return:

- Explanation.
- Key method or structure.
- Worked example or guided reading.
- Common misconception.
- Mini task.
- Recommended next stage.
